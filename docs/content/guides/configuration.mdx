Beever Atlas is configured via environment variables. This is a comprehensive reference of all available configuration options.

## Quick Start

Create a `.env` file in the project root:

```bash
# Minimum required configuration
GOOGLE_API_KEY=your-google-api-key
BEEVER_API_URL=http://localhost:8000
CORS_ORIGINS=http://localhost:5173,http://localhost:3000
```

All other variables have sensible defaults. For production, review the sections below.

## Configuration Categories

<Tabs items={['Core Application', 'Databases', 'LLM Providers', 'Pipeline Settings', 'Integrations', 'QA Agent', 'Security']}>

### Core Application

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `BEEVER_API_URL` | string | `http://localhost:8000` | Public URL of the backend API |
| `CORS_ORIGINS` | string | `http://localhost:5173,http://localhost:3000` | Comma-separated list of allowed CORS origins |
| `BEEVER_API_KEY` | string | `""` | Optional API key for authentication |

**Example**:
```bash
BEEVER_API_URL=https://beever-atlas.example.com
CORS_ORIGINS=https://docs.example.com,https://app.example.com
```

## Configuration File Example

Complete `.env.example`:

```bash
# ============================================================================
# Core Application
# ============================================================================
BEEVER_API_URL=http://localhost:8000
CORS_ORIGINS=http://localhost:5173,http://localhost:3000

# ============================================================================
# Databases
# ============================================================================
WEAVIATE_URL=http://localhost:8080
NEO4J_URI=bolt://localhost:7687
NEO4J_AUTH=neo4j/beever_atlas_dev
MONGODB_URI=mongodb://localhost:27017/beever_atlas
REDIS_URL=redis://localhost:6379

# ============================================================================
# LLM Providers
# ============================================================================
GOOGLE_API_KEY=your-google-api-key-here
JINA_API_KEY=your-jina-api-key-here
TAVILY_API_KEY=your-tavily-api-key-here
LLM_FAST_MODEL=gemini-2.5-flash
LLM_QUALITY_MODEL=gemini-2.5-flash

# ============================================================================
# Pipeline Settings
# ============================================================================
SYNC_BATCH_SIZE=10
SYNC_MAX_MESSAGES=1000
QUALITY_THRESHOLD=0.5
ENTITY_THRESHOLD=0.6
MAX_FACTS_PER_MESSAGE=2

# ============================================================================
# QA Agent
# ============================================================================
QA_CONFIDENCE_THRESHOLD=0.4

# ============================================================================
# Security
# ============================================================================
CREDENTIAL_MASTER_KEY=change-me-to-a-strong-key
```

## Validation

The application validates configuration on startup. Invalid values will cause a startup error with details.

**Test your configuration**:
```bash
python -c "from beever_atlas.infra.config import get_settings; print(get_settings())"
```

## Best Practices

<Callout type="info">
**Security**: Never commit `.env` files to version control. Use `.env.example` as a template.
</Callout>

1. **Use environment-specific files**: `.env.development`, `.env.production`
2. **Document non-default values**: Comment why you changed a value
3. **Validate before deploying**: Test configuration in staging first
4. **Rotate secrets regularly**: Update API keys and passwords
5. **Use strong defaults**: Start secure, relax if needed

## Troubleshooting

**Configuration not loading**: Check file is named `.env` (not `.env.txt`) and is in the project root.

**Invalid values**: Check type matches (integers for numbers, `true`/`false` for booleans).

**Missing required values**: The app will error on startup with the missing variable name.
